# 转换示例：Before & After

本文档展示如何将原始文本转换为符合 Apple 官网风格的 Obsidian 笔记。

### Before (原始文本)

```text

核心概念
协程：使用 async def 定义的函数
事件循环：管理异步任务的执行
Future：表示尚未完成的操作

为什么要使用异步编程
提高性能：可以同时处理多个任务
节省资源：不需要为每个任务创建线程
改善用户体验：应用程序保持响应

如何开始
1. 使用 asyncio 库
2. 定义异步函数
3. 运行事件循环
```

### After (Apple 风格)

```markdown
---
title: Python 异步编程指南
created: 2024-01-25
tags: [python, async, programming, guide]
cssclass: apple-style
---

# Python 异步编程指南

> [!abstract] 概述
> 异步编程让程序在等待耗时操作完成时，不阻塞主线程，从而提高应用的并发性能和响应速度。

## 核心概念

理解异步编程的三个关键组成部分：

### 协程 (Coroutines)

使用 `async def` 定义的函数，可以在执行过程中暂停和恢复。

### 事件循环 (Event Loop)

管理异步任务的执行调度，协调多个协程的运行。

### Future 对象

表示尚未完成的操作，代表一个异步计算的最终结果。

---

## 为什么使用异步编程

> [!note] 核心优势
> 异步编程在现代应用开发中越来越重要，特别是在处理 I/O 密集型任务时。

### 提高性能

程序可以同时处理多个任务，充分利用等待时间。

### 节省资源

不需要为每个任务创建新的线程或进程，降低内存开销。

### 改善用户体验

应用程序在执行耗时操作时保持响应，不会出现界面冻结。

---

## 快速开始

三步上手异步编程：

1. **导入 asyncio 库** - Python 内置的异步编程框架
2. **定义异步函数** - 使用 `async def` 语法
3. **运行事件循环** - 使用 `asyncio.run()` 或 `await`

---

## 注意事项

> [!warning] 常见陷阱
> 避免在异步代码中使用阻塞操作，否则会失去异步编程的优势。

- 不要在异步函数中使用阻塞操作
- 记得 `await` 所有异步调用
- 妥善处理异常情况
- 避免过度嵌套的协程
```

## 转换要点总结

### 清理 (Clean)

- 移除多余空行
- 统一标点符号
- 修正格式不一致

### 结构 (Structure)

- 建立清晰的三级标题层级
- 识别主要内容分类
- 使用 `---` 分隔主要章节

### 增强 (Enhance)

- 添加完整的 YAML frontmatter
- 将重点内容转换为 callouts
- 使用表格展示结构化数据
- 添加任务列表追踪行动项

### 字体 (Font)

- 应用 Apple 系统字体栈
- 确保 SF Pro / PingFang SC 优先
- 使用合适的字号和字重

---

## 关键改进

| 方面 | Before | After |
|------|--------|-------|
| 标题层级 | 无明显层级 | 清晰的 H1-H3 结构 |
| 视觉强调 | 纯文本 | Callouts 突出重点 |
| 元数据 | 缺失 | 完整的 YAML frontmatter |
| 段落间距 | 不一致 | 统一的 1.2em 间距 |
| 章节分隔 | 无 | 使用 `---` 清晰划分 |
| 可读性 | 一般 | 显著提升 |

**转换原则**: 保持内容准确性的同时，提升视觉呈现和阅读体验。
