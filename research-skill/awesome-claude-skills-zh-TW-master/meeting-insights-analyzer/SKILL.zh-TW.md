# 會議見解分析器

這個 skill 將您的會議記錄轉化為關於您溝通模式的可行見解,協助您成為更有效的溝通者和領導者。

## 何時使用此 Skill

- 分析您在多場會議中的溝通模式
- 獲得關於您的領導和引導風格的回饋
- 識別您何時避免困難對話
- 了解您的說話習慣和填充詞
- 追蹤溝通技能的改進情況
- 使用具體範例準備績效評估
- 指導團隊成員的溝通風格

## 此 Skill 的功能

1. **模式識別**：識別跨會議的重複行為，如：
   - 避免衝突或間接溝通
   - 說話比例和輪流
   - 提問與陳述模式
   - 積極傾聽指標
   - 決策方法

2. **溝通分析**：評估溝通效果：
   - 清晰度和直接性
   - 使用填充詞和模糊語言
   - 語調和情緒模式
   - 會議控制和引導

3. **可行的回饋**：提供具體的、帶時間戳的範例，包含：
   - 發生了什麼
   - 為什麼重要
   - 如何改進

4. **趨勢追蹤**：分析多場會議時比較模式隨時間的變化

## 使用方法

### 基本設定

1. 將您的會議記錄下載到資料夾（例如 `~/meetings/`）
2. 在 Claude Code 中導航到該資料夾
3. 要求您想要的分析

### 快速開始範例

```
分析此資料夾中的所有會議，告訴我何時避免衝突。
```

```
查看我過去一個月的會議並識別我的溝通模式。
```

```
比較這兩個會議資料夾之間我的引導風格。
```

### 進階分析

```
分析此資料夾中的所有記錄並：
1. 識別我何時打斷他人
2. 計算我的說話比例
3. 找出我避免給予直接回饋的時刻
4. 追蹤我使用填充詞的情況
5. 展示積極傾聽的良好範例
```

## 指示說明

當使用者請求會議分析時：

1. **發現可用資料**
   - 掃描資料夾尋找記錄檔案（.txt、.md、.vtt、.srt、.docx）
   - 檢查檔案是否包含說話者標籤和時間戳
   - 確認會議的日期範圍
   - 在記錄中識別使用者的姓名/識別符

2. **澄清分析目標**

   如果未指定，詢問他們想要了解什麼：
   - 特定行為（避免衝突、打斷、填充詞）
   - 溝通效果（清晰度、直接性、傾聽）
   - 會議引導技能
   - 說話模式和比例
   - 改進的成長領域

3. **分析模式**

   對於每個請求的見解：

   **避免衝突**：
   - 尋找模糊語言（「也許」、「有點」、「我認為」）
   - 間接措辭而非直接請求
   - 當緊張氣氛出現時轉移話題
   - 沒有承諾的同意（「是的，但是...」）
   - 不解決明顯的問題

   **說話比例**：
   - 計算會議中說話時間的百分比
   - 計算打斷次數（使用者打斷和被打斷）
   - 測量平均說話輪次長度
   - 追蹤問題與陳述的比例

   **填充詞**：
   - 計算「嗯」、「呃」、「像」、「你知道」、「實際上」等
   - 注意每分鐘或每個說話輪次的頻率
   - 識別它們增加的情況（緊張、不確定）

   **積極傾聽**：
   - 參考他人先前觀點的問題
   - 改述或總結他人的想法
   - 建立在他人貢獻的基礎上
   - 提出澄清問題

   **領導與引導**：
   - 決策方法（指令式 vs. 協作式）
   - 如何處理分歧
   - 包含較安靜參與者
   - 時間管理和議程控制
   - 後續跟進和行動項目清晰度

4. **提供具體範例**

   對於找到的每個模式，包含：

   ```markdown
   ### [模式名稱]

   **發現**：[模式的一句話摘要]

   **頻率**：[在 Y 場會議中 X 次]

   **範例**：

   1. **[會議名稱/日期]** - [時間戳]

      **發生了什麼**：
      > [記錄中的實際引用]

      **為什麼這很重要**：
      [解釋影響或錯過的機會]

      **更好的方法**：
      [具體的替代措辭或行為]

   [為 2-3 個最強範例重複]
   ```

5. **綜合見解**

   分析所有模式後，提供：

   ```markdown
   # 會議見解摘要

   **分析期間**：[日期範圍]
   **分析的會議**：[X 場會議]
   **總時長**：[X 小時]

   ## 識別的關鍵模式

   ### 1. [主要模式]
   - **觀察到的**：[您看到的]
   - **影響**：[為什麼重要]
   - **建議**：[如何改進]

   ### 2. [第二個模式]
   [相同結構]

   ## 溝通優勢

   1. [優勢 1 附範例]
   2. [優勢 2 附範例]
   3. [優勢 3 附範例]

   ## 成長機會

   1. **[領域 1]**：[具體、可行的建議]
   2. **[領域 2]**：[具體、可行的建議]
   3. **[領域 3]**：[具體、可行的建議]

   ## 說話統計

   - 平均說話時間：[會議的 X%]
   - 提出的問題：[每場會議平均 X 個]
   - 填充詞：[每分鐘 X 個]
   - 打斷：[每場會議給予 X 次 / 接受 Y 次]

   ## 後續步驟

   [3-5 個改善溝通的具體行動]
   ```

6. **提供後續選項**
   - 在未來會議中追蹤這些相同指標
   - 深入探討特定會議或模式
   - 與產業基準比較
   - 建立個人溝通發展計劃
   - 為績效評估生成摘要

## 範例

### 範例 1：避免衝突分析（靈感來自 Dan Shipper）

**使用者**：「我下載了所有會議錄音並放在一個資料夾中。告訴我所有我巧妙避免衝突的時候。」

**輸出**：
```markdown
# 避免衝突模式

在 15 場會議中找到 23 個實例，您使用間接溝通或避免直接處理緊張關係。

## 模式：對關鍵回饋的模糊

**頻率**：在 7 場會議中 8 次

**範例 1：與 Sarah 的 1:1** - 00:14:32

**發生了什麼**：
> 「所以，我在想...也許我們可以，像是，可能考慮再看一下時間表？
> 我的意思是，如果您認為這有道理。但無論您認為什麼最好！」

**為什麼這很重要**：
您需要解決 Sarah 的專案進度落後，但模糊的語言（「也許」、「可能」、「如果您認為」）
和最後的轉移（「無論您認為什麼」）讓她很容易錯過緊急性。

**更好的方法**：
「Sarah，專案落後兩週。我們需要討論阻礙進度的原因
並今天建立新的時間表。」

[其他範例...]

## 建議

1. **在第一句話中直接說出問題**
2. **移除模糊詞語**，如「也許」、「有點」、「某種程度上」
3. **提出具體問題**而非暗示
4. **安排困難對話**而非隨意提出
```

### 範例 2：領導引導

**使用者**：「分析我的團隊會議並告訴我關於我的引導風格。」

**輸出**：提供以下見解：
- 您說話與團隊成員的比例（60% vs. 40%）
- 您是提問還是陳述（3:1 比例）
- 您如何處理分歧（傾向於過快解決）
- 誰說話最少以及您是否吸引他們
- 良好和錯過的引導時刻範例

### 範例 3：個人發展追蹤

**使用者**：「比較我第一季與第二季的會議，看我是否改善了傾聽技能。」

**輸出**：建立比較分析，顯示：
- 打斷減少（每場會議 8 次 → 3 次）
- 澄清問題增加（每場會議 2 個 → 7 個）
- 建立在他人想法上的改進
- 顯示差異的具體範例
- 仍需成長的領域

## 設定技巧

### 獲取會議記錄

**從 Granola**（Lenny's newsletter 訂閱免費）：
- Granola 自動轉錄您的會議
- 將記錄匯出到資料夾：[如何操作的說明]
- 將 Claude Code 指向該資料夾

**從 Zoom**：
- 啟用帶轉錄的雲端錄製
- 會議後下載 VTT 或 SRT 檔案
- 儲存在專用資料夾中

**從 Google Meet**：
- 使用 Google Docs 自動轉錄
- 將記錄文件儲存到資料夾
- 下載為 .txt 檔案或授予 Claude Code 存取權限

**從 Fireflies.ai、Otter.ai 等**：
- 批量匯出記錄
- 儲存在本地資料夾
- 對資料夾執行分析

### 最佳實踐

1. **一致的命名**：使用 `YYYY-MM-DD - 會議名稱.txt` 格式
2. **定期分析**：每月或每季審查趨勢
3. **具體查詢**：一次詢問一種行為以獲得深度
4. **隱私**：保持敏感會議資料在本地
5. **行動導向**：一次專注於一個改進領域

## 常見分析請求

- 「我何時避免困難對話？」
- 「我多常打斷他人？」
- 「我的說話與傾聽比例是多少？」
- 「我提出好問題嗎？」
- 「我如何處理分歧？」
- 「我是否包容所有聲音？」
- 「我是否使用太多填充詞？」
- 「我的行動項目有多清晰？」
- 「我是否遵守議程還是偏離主題？」
- 「我的溝通隨時間如何變化？」
