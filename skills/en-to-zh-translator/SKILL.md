# 英中技术翻译器

此技能提供专业的英中翻译服务，专门针对技术和编程内容进行定制。

## 核心翻译原则

### 何时使用此技能
- 翻译 AI 提示和指令
- 将技术文档转换为中文
- 翻译编程教程和指南
- 转换代码注释和技术说明
- 翻译以开发者为重点的内容

### 翻译指南

#### 1. 保持技术完整性
- **代码块**：保持所有代码不变，保留语法高亮
- **技术术语**：在适当时保留括号中的原始英文术语
- **变量名**：不要翻译函数名、变量名或 API
- **文件扩展名**：保持文件扩展名和路径不变
- **数学表达式**：保持所有 LaTeX 数学表达式不变

#### 2. 格式保持
- **Markdown 结构**：保持标题、列表、粗体、斜体格式
- **链接**：保持 URL 和技术参考不变
- **代码格式**：保持内联代码格式（`反引号`）
- **表格**：保持表格结构和技术内容

#### 3. 翻译质量
- **技术准确性**：确保技术概念的精确翻译
- **专业语调**：使用正式的技术中文语言
- **一致性**：保持术语的一致性
- **上下文意识**：考虑编程和技术上下文

## 翻译模式

### 代码相关内容
```markdown
**原文：**
```rust
fn calculate_fibonacci(n: u32) -> u64 {
    // Calculate fibonacci number
}
```

**翻译：**
```rust
fn calculate_fibonacci(n: u32) -> u64 {
    // 计算斐波那契数
}
```

### 技术指令
**原文：**
> Create a function that accepts a `u32` parameter and returns a `Result<T, Error>`

**翻译：**
> 创建一个接受 `u32` 参数并返回 `Result<T, Error>` 的函数

### 数学表达式
**原文：**
> Calculate the $n$-th Fibonacci number using $O(n)$ time complexity

**翻译：**
> 使用 $O(n)$ 时间复杂度计算第 $n$ 个斐波那契数

## 常见技术术语映射

### 编程概念
- function → 函数
- variable → 变量
- parameter → 参数
- return value → 返回值
- error handling → 错误处理
- algorithm → 算法
- data structure → 数据结构

### 开发流程
- implementation → 实现
- optimization → 优化
- debugging → 调试
- testing → 测试
- deployment → 部署
- version control → 版本控制

## 特殊处理

### AI 提示
对于 AI 提示翻译：
- 翻译角色描述和指令
- 保留技术要求和约束
- 保持输出格式规范
- 保持示例结构完整

### 文档
对于技术文档：
- 翻译说明文字，同时保留代码示例
- 保持逐步程序格式
- 保留警告和注释部分
- 保持参考表和技术规范

## 输出格式

始终提供：
1. 源内容的**直接翻译**
2. 保持所有技术元素不变的**保留格式**
3. 保持技术准确性的**中文翻译**
4. 具有适当中文语言流畅度的**原始结构**
5. 覆盖原始文件或是选中内容

## 质量保证

在最终确定翻译之前，请验证：
- 所有代码块保持不变
- 技术术语翻译正确
- 数学表达式被保留
- 格式结构得到保持
- 中文文本自然且专业
