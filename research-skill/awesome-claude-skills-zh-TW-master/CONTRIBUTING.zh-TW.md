# 為 Awesome Claude Skills 貢獻

感謝您有興趣為頂尖的 Claude Skills 集合做出貢獻！本指南將協助您新增對整個 Claude 社群有益的新技能。

## 開始之前

- 確保您的技能基於**真實使用案例**，而非假設情境。
- 搜尋現有技能以避免重複。
- 如果可能，請註明使用案例的原始人員或來源。

## 技能要求

所有技能必須：

1. **解決真實問題** - 基於實際使用，而非理論應用。
2. **文件完整** - 包含清楚的說明、範例和使用案例。
3. **易於存取** - 在可能的情況下為非技術使用者撰寫。
4. **包含範例** - 展示實用的真實世界使用方式。
5. **經過測試** - 驗證技能在 Claude.ai、Claude Code 和/或 API 上能夠運作。
6. **安全** - 在破壞性操作之前確認。
7. **可移植** - 在適用的情況下跨 Claude 平台運作。

## 技能結構

使用您的技能名稱建立新資料夾（使用小寫和連字號）：

```
skill-name/
└── SKILL.md
```

## 將您的技能新增至 README

1. 選擇適當的類別：
   - 商業與行銷
   - 溝通與寫作
   - 創意與媒體
   - 開發
   - 生產力與組織

2. 在類別內按字母順序新增您的技能：

```markdown
- [Skill Name](./skill-name/) - 一句話描述。靈感來自 [人員/來源]。
```

3. 完全遵循現有格式 - 無表情符號，標點符號一致。

## Pull Request 流程

1. Fork 此儲存庫
2. 建立分支：`git checkout -b add-skill-name`
3. 新增您的技能資料夾與 SKILL.md
4. 在適當的類別中更新 README.md 以包含您的技能
5. 提交您的變更：`git commit -m "Add [Skill Name] skill"`
6. 推送至您的 fork：`git push origin add-skill-name`
7. 開啟 Pull Request

## Pull Request 指南

您的 PR 應該：

- **標題**："Add [Skill Name] skill"
- **描述**：說明真實世界的使用案例並包含：
  - 它解決什麼問題
  - 誰使用此工作流程
  - 歸屬/靈感來源
  - 如何使用的範例

## 行為準則

- 保持尊重和建設性
- 註明原始來源和靈感
- 專注於實用、有幫助的技能
- 撰寫清楚、易於理解的文件
- 在提交之前測試您的技能

## 有問題嗎？

如果您對貢獻有疑問或需要協助建構您的技能，請開啟 issue。

## 歸屬

當新增基於某人的工作流程或使用案例的技能時，請包含適當的歸屬：

```markdown
**靈感來源：**[Person Name] 的工作流程
```

或

```markdown
**致謝：**基於 [Company/Team] 的流程
```

範例：
- **靈感來源：**Dan Shipper 的會議分析工作流程
- **靈感來源：**Teresa Torres 的內容研究流程
- **致謝：**基於 Notion 的文件工作流程

## 技能類別

### 商業與行銷
用於潛在客戶開發、競爭研究、品牌建立和業務發展的技能。

### 溝通與寫作
用於改善溝通、分析對話和建立內容的技能。

### 創意與媒體
用於處理圖像、影片、音訊和創意內容的技能。

### 開發
用於軟體開發、文件和技術工作流程的技能。

### 生產力與組織
用於組織檔案、管理任務和個人生產力的技能。
