# 潛在客戶研究助手

這個 skill 透過分析您的產品/服務、了解您的理想客戶檔案，並提供可行的拓展策略，協助您識別和評估業務的潛在客戶。

## 何時使用此 Skill

- 為您的產品/服務尋找潛在客戶或客戶
- 建立要聯繫的合作公司清單
- 識別銷售拓展的目標帳戶
- 研究符合您理想客戶檔案的公司
- 準備業務開發活動

## 此 Skill 的功能

1. **了解您的業務**：分析您的產品/服務、價值主張和目標市場
2. **識別目標公司**：根據以下條件尋找符合您理想客戶檔案的公司：
   - 產業和部門
   - 公司規模和位置
   - 他們使用的技術堆疊和工具
   - 成長階段和資金
   - 您的產品解決的痛點
3. **優先排序潛在客戶**：根據匹配分數和相關性對公司進行排名
4. **提供聯繫策略**：建議如何以個人化訊息接觸每個潛在客戶
5. **豐富資料**：收集有關決策者和公司背景的相關資訊

## 使用方法

### 基本用法

只需描述您的產品/服務和您要尋找的內容：

```
我正在建構 [產品描述]。在 [位置/產業] 中為我找到 10 家
適合這個產品的公司。
```

### 搭配您的程式碼庫

為了獲得更好的結果，從您產品的原始程式碼目錄執行：

```
查看我在這個儲存庫中建構的內容，並識別 [位置/產業] 中
將受益於此產品的前 10 家公司。
```

### 進階用法

進行更有針對性的研究：

```
我的產品：[描述]
理想客戶檔案：
- 產業：[產業]
- 公司規模：[規模範圍]
- 位置：[位置]
- 目前痛點：[痛點]
- 他們使用的技術：[技術堆疊]

為我找到 20 個合格的潛在客戶，每個都附上聯繫策略。
```

## 指示說明

當使用者請求潛在客戶研究時：

1. **了解產品/服務**
   - 如果在程式碼目錄中，分析程式碼庫以了解產品
   - 詢問有關價值主張的澄清問題
   - 識別關鍵功能和優勢
   - 了解它解決的問題

2. **定義理想客戶檔案**
   - 確定目標產業和部門
   - 識別公司規模範圍
   - 考慮地理偏好
   - 了解相關痛點
   - 注意任何技術要求

3. **研究並識別潛在客戶**
   - 搜尋符合條件的公司
   - 尋找需求信號（職位公告、技術堆疊、最新新聞）
   - 考慮成長指標（資金、擴張、招聘）
   - 識別具有互補產品/服務的公司
   - 檢查預算指標

4. **優先排序和評分**
   - 為每個潛在客戶建立匹配分數（1-10）
   - 考慮因素包括：
     - 與理想客戶檔案的對齊
     - 即時需求信號
     - 預算可用性
     - 競爭格局
     - 時機指標

5. **提供可行的輸出**

   對於每個潛在客戶，提供：
   - **公司名稱**和網站
   - **為何他們是好的匹配**：基於他們業務的具體原因
   - **優先級分數**：1-10 附解釋
   - **決策者**：目標角色/職稱（例如「工程副總裁」）
   - **聯繫策略**：個人化方法建議
   - **價值主張**：您的產品如何解決他們的特定問題
   - **對話開場白**：在拓展中提及的具體要點
   - **LinkedIn URL**：如果可用，便於連接

6. **格式化輸出**

   以清晰、可掃描的格式呈現結果：

   ```markdown
   # 潛在客戶研究結果

   ## 摘要
   - 找到的潛在客戶總數：[X]
   - 高優先級（8-10）：[X]
   - 中優先級（5-7）：[X]
   - 平均匹配分數：[X]

   ---

   ## 潛在客戶 1：[公司名稱]

   **網站**：[URL]
   **優先級分數**：[X/10]
   **產業**：[產業]
   **規模**：[員工數/營收範圍]

   **為何他們是好的匹配**：
   [基於他們業務的 2-3 個具體原因]

   **目標決策者**：[角色/職稱]
   **LinkedIn**：[URL（如果可用）]

   **針對他們的價值主張**：
   [對此公司的具體優勢]

   **拓展策略**：
   [個人化方法 - 提及具體痛點、最近的公司新聞或相關背景]

   **對話開場白**：
   - [具體要點 1]
   - [具體要點 2]

   ---

   [為每個潛在客戶重複]
   ```

7. **提供後續步驟**
   - 建議將結果儲存為 CSV 以匯入 CRM
   - 提議起草個人化的拓展訊息
   - 根據時機建議優先順序
   - 建議對頂級潛在客戶進行後續研究

## 範例

### 範例 1：來自 Lenny's Newsletter

**使用者**：「我正在建構一個在 AI 程式碼助手查詢中遮罩敏感資料的工具。尋找潛在客戶。」

**輸出**：建立符合以下條件的公司優先清單：
- 使用 AI 程式碼助手（Copilot、Cursor 等）
- 處理敏感資料（金融科技、醫療保健、法律）
- 在其 GitHub 儲存庫中有使用程式碼代理的證據
- 可能在程式碼中意外暴露敏感資料
- 包含相關決策者的 LinkedIn URL

### 範例 2：本地業務

**使用者**：「我經營一家遠端團隊生產力顧問公司。在灣區為我找到 10 家最近轉為遠端的公司。」

**輸出**：識別符合以下條件的公司：
- 最近發布遠端職位
- 宣布遠端優先政策
- 正在招聘分散式團隊
- 顯示遠端工作挑戰的跡象
- 為每家公司提供個人化的拓展策略

## 獲得最佳結果的技巧

- **具體說明**您的產品及其獨特價值
- **從您的程式碼庫執行**（如適用）以獲取自動背景資訊
- **提供背景資訊**關於您的理想客戶檔案
- **指定限制條件**，如產業、位置或公司規模
- **要求後續**研究有前景的潛在客戶以獲得更深入的見解
