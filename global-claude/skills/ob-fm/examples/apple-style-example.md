# Apple Style Formatter 示例

这是一个完整的 Apple 风格 Markdown 排版示例，展示了如何应用 Apple 设计语言来创建优雅、易读的文档。

## 设计原则

Apple 设计语言遵循三个核心原则：简洁、清晰、精致。这些原则体现在每一个排版细节中。

### 简洁

去除所有不必要的装饰，让内容本身成为焦点。使用克制的配色方案，主要使用灰度系统。

### 清晰

通过精确的排版层级、充足的留白和恰当的间距，确保信息传达清晰明了。

### 致

每个细节都经过精心设计，从字重到间距，从颜色到圆角，都追求完美的视觉效果。

---

## 标题系统

Apple 风格采用简洁的三级标题系统，通过字号、字重和装饰元素来实现清晰的视觉层级。

# H1 主要章节标题

H1 是最高级别的标题，字号为 28px，字重 700，下方有细细的浅灰色下划线。用于文档的主要章节。

## H2 次级章节标题

H2 是次级标题，字号为 21px，字重 600，左侧有 3px 宽的深灰色边框。用于章节下的主要内容。

### H3 小节标题

H3 是第三级标题，字号为 18px，字重 500，没有任何装饰，完全靠字号和字重来区分。用于细分的子内容。

---

## 段落和间距

Apple 风格的段落排版采用书籍风格标准，提供舒适的阅读体验。

### 普通段落

正文采用 1.8 行高和 0.02em 字间距，接近纸质书的阅读感受。段落之间使用 12px 间距，保持紧凑但不拥挤的效果。

这个段落展示了标准正文样式。使用 `#1d1d1f` 近黑色作为文字颜色，确保最佳的可读性。行高设置为 1.8，给文字充足的呼吸空间。

### 段落分组

使用三个连字符 `---` 可以创建段落组分隔符。这个分隔符本身是不可见的，但它会在前后各产生 24px 的间距，用于标记新的主题或内容组。

---

这是一个新的段落组，与上一段之间有更大的间距（24px）。这种设计帮助读者理解文档的结构层次。

## 引用块

引用块使用浅灰色背景和左侧边框，创建柔和的视觉效果。

> 这是一个标准的引用块。它具有浅灰色的背景（#f5f5f7）和左侧的 3px 浅灰色边框。文字颜色使用中灰色（#6e6e73），与正文形成对比。
>
> 引用块支持多行内容。每行都会自动应用相同的样式。右侧有 4px 的圆角，增加视觉柔和度。

## 代码展示

### 行内代码

行内代码使用 `monospace` 字体，粉红色文字（#c7254e），浅灰色背景，和圆角边框。例如：`const greeting = "Hello, Apple Style!"`

### 代码块

代码块使用等宽字体栈，浅灰背景（#fafafa），带有边框和圆角：

```python
def apple_style_formatter(content: str) -> str:
    """
    将内容格式化为 Apple 风格的 Markdown

    Args:
        content: 原始 Markdown 内容

    Returns:
        格式化后的 Markdown 内容
    """
    # 应用 Apple 设计系统
    formatted = apply_heading_hierarchy(content)
    formatted = adjust_spacing(formatted)
    formatted = apply_color_system(formatted)

    return formatted

# 使用示例
result = apple_style_formatter(document)
```

## 颜色系统示例

Apple 风格的配色遵循"灰色管结构，蓝色管交互"的原则：

### 灰度系统（结构元素）

- **主文本** (#1d1d1f)：用于正文和标题
- **次要文本** (#6e6e73)：用于辅助说明和 H2 边框
- **三级文本** (#86868b)：用于更次要的文字
- **分隔线** (#d2d2d7)：用于 H1 下划线和引用边框

### 强调色（交互元素）

- **链接蓝** (#0071e3)：仅用于链接

> **设计原则**：整个文档中只使用一个彩色——链接蓝。所有其他元素都使用灰度系统。这种克制的配色确保内容成为焦点。

## 字体系统

Apple 风格优先使用系统字体栈：

### 正文和标题

优先使用 San Francisco 字体，依次回退到 Helvetica Neue、PingFang SC（苹方）、Hiragino Sans GB（冬青黑）、Arial。

这段文字使用的是您系统中的 Apple 系统字体。如果您在 Mac 或 iOS 设备上阅读，看到的就是 SF Pro 字体。

## 圆角系统

柔和的圆角设计增加视觉亲和力：

- **小圆角 (4px)**：行内代码、小按钮
- **中圆角 (8px)**：代码块、卡片
- **大圆角 (12px)**：面板、大型容器
- **特大圆角 (16px)**：特殊元素

## 最佳实践

### ✅ 推荐做法

1. **保持三级标题结构**：不要使用超过三级的标题层级
2. **应用一致的间距**：使用 8px 基准间距系统
3. **遵循配色原则**：灰色用于结构，蓝色仅用于链接
4. **使用系统字体**：优先使用 Apple 系统字体栈
5. **保持行高一致**：正文使用 1.8 行高

### ❌ 避免做法

1. 不要使用超过三个标题级别
2. 不要添加除蓝色以外的彩色元素
3. 不要使用不规则的间距值
4. 不要过度装饰内容
5. 不要混合多种设计系统

关键要素包括：

- **三级标题系统**：清晰的层级结构

- **克制配色方案**：内容优先的设计理念
- **系统字体栈**：原生的视觉效果
- **8px 基准间距**：一致的视觉节奏

将这些原则应用到您的文档中，即可获得专业级的 Apple 风格排版效果。
