# Changelog Generator

此技能將技術性的 git 提交轉換為精緻、使用者友善的變更日誌，讓您的客戶和使用者真正理解和欣賞。

## 何時使用此技能

- 為新版本準備發布說明
- 建立每週或每月產品更新摘要
- 為客戶記錄變更
- 撰寫應用程式商店提交的變更日誌條目
- 生成更新通知
- 建立內部發布文件
- 維護公開變更日誌/產品更新頁面

## 此技能的功能

1. **掃描 Git 歷史**：分析特定時間段或版本之間的提交
2. **分類變更**：將提交分組為邏輯類別（功能、改進、錯誤修復、破壞性變更、安全性）
3. **轉換技術 → 使用者友善**：將開發者提交轉換為客戶語言
4. **專業格式設定**：建立乾淨、結構化的變更日誌條目
5. **過濾雜訊**：排除內部提交（重構、測試等）
6. **遵循最佳實踐**：應用變更日誌指南和您的品牌聲音

## 如何使用

### 基本使用

從您的專案儲存庫：

```
從上次發布以來的提交建立變更日誌
```

```
為過去一週的所有提交生成變更日誌
```

```
為版本 2.5.0 建立發布說明
```

### 使用特定日期範圍

```
為 3 月 1 日到 3 月 15 日之間的所有提交建立變更日誌
```

### 使用自訂指南

```
為自 v2.4.0 以來的提交建立變更日誌，使用我的 CHANGELOG_STYLE.md
中的變更日誌指南
```

## 範例

**使用者**："為過去 7 天的提交建立變更日誌"

**輸出**：
```markdown
# Updates - Week of March 10, 2024

## ✨ 新功能

- **團隊工作區**：為不同專案建立獨立的工作區。邀請團隊成員並保持一切井然有序。

- **鍵盤快速鍵**：按 ? 查看所有可用的快速鍵。無需觸摸滑鼠即可更快地導航。

## 🔧 改進

- **更快的同步**：檔案現在在設備間同步速度提高 2 倍
- **更好的搜尋**：搜尋現在包括檔案內容，而不僅僅是標題

## 🐛 修復

- 修復了大型圖片無法上傳的問題
- 解決了排程貼文中的時區混淆
- 修正了通知徽章計數
```

**靈感來源：**Manik Aggarwal 在 Lenny's Newsletter 的使用案例

## 提示

- 從 git 儲存庫根目錄執行
- 指定日期範圍以獲得聚焦的變更日誌
- 使用您的 CHANGELOG_STYLE.md 以實現一致的格式設定
- 在發布之前審查和調整生成的變更日誌
- 直接將輸出儲存到 CHANGELOG.md
